## Тесты от ИСП РАН для открытого этапа испытаний статических анализаторов

В архиве находятся тесты для следующих языков программирования:
* C99 и C++11: подпапка `c`;
* C# 12: подпапка `cs`;
* Java 21: подпапка `java`;
* Go 1.21: подпапка `go`;
* Python 3.12: подпапка `python`;
* JavaScript ES6: подпапка `cs`;

Для удобства тесты разделены по подпапкам, согласно общим критическим типам дефектов:
* `1-tainted`: ошибки непроверенного использования чувствительных данных;
* `2-integer-overflow`: ошибки целочисленного переполнения и некорректного совместного использования знаковых и беззнаковых чисел;
* `3-buffer-overflow`: ошибки переполнения буфера (записи или чтения за пределами выделенной для буфера памяти);
* `4-sec-api-misuse`: ошибки некорректного использования системных процедур и интерфейсов, связанных с обеспечением информационной безопасности (шифрования, разграничения доступа и пр.);

А также специфичным для определённых языков:
* `lang1-null-deref`: ошибки разыменования нулевого указателя;
* `lang2-div-by-zero`: ошибки деления на ноль;
* `lang3-mem-mgmt`: ошибки управления динамической памятью (выделения, освобождения, использования освобожденной памяти);
* `lang4-format-args`: ошибки использования форматной строки;
* `lang5-uninit`: ошибки использования неинициализированных переменных;
* `lang6-mem-leak`: ошибки утечек памяти, незакрытых файловых дескрипторов и дескрипторов сетевых соединений.

Каждый тест состоит из одного файла и может быть проанализирован независимо. Дополнительные зависимости, кроме предоставляемых стандартными библиотеками соответствующих языков, не используются. Для компиляции тестов, где это применимо, используются стандартные команды стандартных компиляторов без дополнительных опций. Каждый тест либо содержит единственный дефект соответствующего типа (тест на истинные срабатывания), либо построен так, чтобы не содержать дефекта (тест на ложные срабатывания). В коде каждого теста содержатся комментарии для облегчения понимания структуры теста и дефекта, который он содержит или не должен содержать.

В каждой папке вместе с исходным кодом тестов хранится манифест `manifest.toml` с записями для каждого из тестов с указанием:
* примерного типа дефекта в классификации CWE, на который данный тест рассчитан;
* строки кода, где ожидается выдача предупреждения анализатором;
* набора тегов для дополнительной информации о свойствах теста, например, какие свойства анализа могут быть полезны при поиска в нём дефекта.

Пример записи в манифесте:
```
[["test_257_cwe120_Buffer-Overflow_exception-any.cpp"]]
lines = [ 31 ]
rule = "CWE120"
tags = [ "exception", "flow", ]
```

Данная запись содержит информацию о том, что в файле `test_257_cwe120_Buffer-Overflow_exception-any.cpp` на строке `31` ожидается предупреждение о переполнении буфера с классификатором CWE120 или связанного с ним, и в данном тесте для анализа может потребоваться обработка исключений и чувствительность к потоку.

Если в записи о тесте отсутствует поле `lines` или данный список пустой, то в данном тесте не предполагается обнаружение дефекта соответствующего типа.
